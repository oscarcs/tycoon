Multiplayer progress

- Added a multiplayer session model to GameState with solo, host, and joined modes.
- Added invite-link creation from the current seed/world id.
- Added a Cloudflare Worker and Durable Object in worker/src/index.ts with WebSocket Hibernation via acceptWebSocket, per-world routing with getByName, SQLite-backed patch persistence, and peer rebroadcast.
- Added state patch encoding for synchronizing clock, company, stations, trains, and seed-derived world identity.
- Added wrangler.jsonc with TYCOON_WORLD Durable Object binding and v1 SQLite migration.
- Added optional browser WebSocket client hookup through VITE_TYCOON_SYNC_URL.

Implementation boundary

- The browser game can create and display share links now and will connect to a deployed/local Worker when VITE_TYCOON_SYNC_URL is configured.
- The Worker can be run with npm run worker:dev or deployed with npm run worker:deploy after Cloudflare authentication is available.
